
# Troubleshooting - MyBatis

<br>

## 게시글 상세 조회 구조 개선

### 문제
게시글 상세와 댓글 목록을 `association`, `collection`을 이용해
하나의 JOIN 쿼리로 조회하려 했다.

하지만 댓글 수만큼 게시글 데이터가 반복되고,
댓글 페이징과 비동기 갱신을 적용하기 어려웠다.

### 검토
`collection`의 `select` 속성을 이용한 중첩 조회도 고려했다.

다만 댓글 조회가 MyBatis 매핑 내부에 결합되고,
향후 댓글만 독립적으로 조회하거나 갱신할 때 별도 메서드가 다시 필요하다고 판단했다.

### 해결
게시글 상세와 댓글 목록을 각각 별도의 Mapper 메서드로 조회하고,
Service 계층에서 하나의 응답 DTO로 조합하도록 변경했다.

### 결과
- 게시글 데이터 중복 감소
- 댓글 페이징 적용 용이
- 댓글 영역의 비동기 갱신 가능
